**What this demo covers**

This demonstration shows how Red Hat OpenShift serves as a comprehensive application platform that streamlines developer workflows, automates CI/CD, secures the software supply chain, and enables AI-enhanced applications. Through the Parasol Insurance scenario, presenters follow a continuous story across three sections that build on each other.

**Preparation**

* xref:01-overview.adoc[Overview and presenter preparation]
* xref:02-details.adoc[Environment details and setup]

**Section 1: Foundational application platform**

Application modernization, standardized development with DevSpaces, Quarkus dev mode, Tekton pipelines, Argo CD delivery, and platform operations with Service Mesh and Kiali.

* xref:03-module-01-developer-experience.adoc[Module 1: Developer experience] — MTA migration context, DevSpaces with devfile, Quarkus dev mode, and platform engineer governance
* xref:04-module-02-cicd-pipeline.adoc[Module 2: CI/CD pipeline] — Tekton pipeline triggering, catching missing tests with AI-assisted fixes, and GitOps delivery with Argo CD
* xref:05-module-03-platform-operations.adoc[Module 3: Platform operations] — Traffic management with Service Mesh, observability with Kiali, and broader platform capabilities

**Section 2: Advanced developer services**

Red Hat Developer Hub for catalog and self-service, secure build pipeline with ACS and Tekton Chains, Vault for secrets, and trusted software supply chain with TPA.

* xref:06-module-04-developer-hub.adoc[Module 4: Developer Hub and catalog] — Developer Lightspeed for MTA, RHDH catalog exploration, and self-service component templating
* xref:07-module-05-secure-development.adoc[Module 5: Secure development workflow] — Dependency Analytics in DevSpaces, secure build pipeline with signing and attestation, and external secrets with Vault
* xref:08-module-06-supply-chain.adoc[Module 6: Trusted software supply chain] — Production deployment with admission control, SBOM management with TPA, and application topology

**Section 3: Intelligent applications**

Leveraging existing LLM endpoints to enhance applications, using the same golden path patterns from Sections 1 and 2.

* xref:09-module-07-ai-applications.adoc[Module 7: AI-enhanced applications] — Building an AI component with DevHub templates, connecting to LLM and Kafka, and deploying through the trusted pipeline

**Target audience**

Enterprise architects, engineering managers, IT decision makers, and line-of-business owners responsible for developer and platform engineering teams.

**Demo duration**

Each section is approximately 45 minutes (15 minutes per module). Sections can be presented independently or combined for a comprehensive platform story.
